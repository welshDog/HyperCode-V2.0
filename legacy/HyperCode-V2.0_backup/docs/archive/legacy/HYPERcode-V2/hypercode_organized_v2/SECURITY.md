# Security Policy

## Supported Versions

Only the latest version of HyperCode receives security updates. We recommend always using the most recent release.

## Reporting a Vulnerability

### Private Disclosure

Please report security issues to `security@hypercode.dev`. We will respond within 48 hours.

### Public Disclosure

For non-critical issues, please open a GitHub issue with the "security" label.

## Security Updates

- Security updates are released as patch versions (e.g., 1.0.x)
- Critical security fixes may receive out-of-band updates

## Security Best Practices

1. Always verify package hashes before installation
2. Use virtual environments
3. Regularly update dependencies

## Known Security Considerations

- HyperCode executes AI-generated code in a sandboxed environment
- API keys should be managed through environment variables
- All data processing happens locally by default
