## Formatting Standards
- Headings: Title Case, short and descriptive.
- Bullets: concise one-liners with clear actions.
- Code blocks: fenced with language tags; keep examples minimal.
- Links: use absolute file links for quick navigation.

## Style Guidelines
- Clear, professional language; avoid jargon or explain it briefly.
- Step-by-step instructions for complex procedures.
- Include examples where helpful; prefer small, runnable snippets.

## Maintenance Schedule
- Weekly: review CI status and update docs impacted by changes.
- Monthly: refresh dependency and security sections; validate deployment steps.
- Quarterly: revisit architecture diagrams and platform overviews.
- Ownership: Documentation stewards (Backend, Frontend, DevOps) rotate monthly.
