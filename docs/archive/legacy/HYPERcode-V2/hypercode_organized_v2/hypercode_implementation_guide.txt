### Day 2: Content Creation (4-6 hours)

#### Morning: Research Foundation (2-3 hours)

**Task 2.1: Add Core Research Notes**

Create these essential research notes (pick 5-7 most relevant):

1. **Cognitive Science**
   - `research/cognitive_science/working-memory-adhd.md`
   - `research/cognitive_science/executive-function-adhd.md`
   - `research/cognitive_science/pattern-recognition-autism.md`

2. **Accessibility**
   - `research/accessibility/visual-noise-reduction.md`
   - `research/accessibility/dyslexia-friendly-fonts.md`

3. **Language Comparisons**
   - `research/language_comparisons/python-readability-analysis.md`
   - `research/language_comparisons/rust-error-messages-analysis.md`

**Pro Tip:** Start with abstracts/summaries. You can expand later. The goal is to establish the graph structure.

**Task 2.2: Validate After Each File**
```bash
# After adding each file:
python validate_research_db.py .

# Fix any errors immediately (easier than fixing 10 files at once)
```

#### Afternoon: Decisions (2-3 hours)

**Task 2.3: Document Key Decisions**

Create 2-3 architecture decision records:

1. **ADR-001**: Core syntax philosophy (word-based vs symbols)
2. **ADR-002**: Error message framework
3. **ADR-003**: Type system approach OR nesting limits

For each:
1. Copy `_templates/decision.md`
2. Fill out Context, Decision, Rationale
3. Link to at least 2 research notes in Relations section
4. Include code examples

**Task 2.4: Link Research ‚Üî Decisions**

Go back to research notes and add Relations links:
```markdown
## Relations

### Supports
- [[adr-001-syntax-minimalism]] - Provides evidence for this decision
- [[adr-003-nesting-limits]] - Informs this design choice
```

**Task 2.5: Validate Complete System**
```bash
python validate_research_db.py .
```

You should now have:
- ‚úÖ 5-7 research notes
- ‚úÖ 2-3 design decisions
- ‚úÖ Bidirectional links between them
- ‚úÖ Zero validation errors

## üêõ Troubleshooting

### "Validation failing with YAML errors"
**Fix:** Check indentation (use spaces, not tabs). Ensure closing `---` exists.

### "Missing Relations section"
**Fix:** Every file must have `## Relations` heading, even if empty:
```markdown
## Relations
None yet - foundational document.
```

### "Broken links [[file-not-found]]"
**Fix:** Either:
1. Create the missing file, or
2. Remove the link temporarily

### "CI not triggering"
**Fix:** Check `.github/workflows/` path is correct. Push to a branch and open PR.
